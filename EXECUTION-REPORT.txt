🚀 下一步 - 选择执行方式
════════════════════════════════════════════════════

方式 A: 使用 Git Bundle（最推荐）
────────────────────────────────────────────

操作：下载 crypto-dashboard.bundle 到本地

步骤：
   1. 下载 bundle 文件到本地电脑

   2. 在本地电脑执行：
      git clone crypto-dashboard.bundle crypto-dashboard
      cd crypto-dashboard

   3. 生成 GitHub Token
      访问: https://github.com/settings/tokens
      创建 Token (classic)，勾选 repo 权限

   4. 推送到 GitHub
      export GITHUB_TOKEN=your_token_here
      git remote add origin https://$GITHUB_TOKEN@github.com/YOUR_USERNAME/crypto-dashboard.git
      git push -u origin main

   5. 在浏览器连接 Cloudflare Pages
      https://dash.cloudflare.com/ → Workers & Pages → Create → Connect to Git
      选择仓库 → 配置 build → 完成

优点：单文件传输，本地操作灵活，完全控制

方式 B: 设置环境变量后执行脚本
────────────────────────────────────────────

操作：生成 GitHub Token 重新执行

步骤：
   1. 生成 GitHub Token（同方式 A 第 3 步）

   2. 在服务器执行：
       cd /root/.openclaw/workspace/crypto-dashboard
       export GITHUB_TOKEN=your_token_here
       export GITHUB_USERNAME=your_github_username
       ./auto-exec.sh

   3. 如果成功，按提示在浏览器连接 Cloudflare

优点：自动化程度高，只需提供 Token

方式 C: 使用 GitHub CLI（需要交互）
────────────────────────────────────────────

如果服务器已安装 GitHub CLI (gh)：

步骤：
   1. gh auth login（会有交互式流程）

   2. gh repo create crypto-dashboard --public

   3. git remote add origin $(gh repo view --json url -q .url)

   4. git push -u origin main

优点：自动创建仓库

注意：服务器环境可能不支持完整交互
